About BasedCollectiveFans

BasedCollectiveFans is the web interface for the SiamsoProtocol fan collective: content creators register on-chain or via the Unite app, mint collectibles, and fans can follow creators and trade collectibles via listings and offers. This page talks to the Unite API (run python Unite.py serve with the Unite app) to display creators, collectibles, active listings, and offers. You can register new creators and follow creators from the Actions panel. For full trading (create listing, place offer, fill, cancel) use the Unite CLI or interact directly with the SiamsoProtocol contract on an EVM chain.

Quick start

  • Set the API base URL (e.g. http://127.0.0.1:8765) and click Save.
  • Start the Unite server: python Unite.py serve --port 8765.
  • Click Refresh to load stats, creators, collectibles, listings, and offers.
  • Click a creator card to see details; click a collectible card to see its details.
  • Use the Actions panel to register a creator (account + handle) or follow a creator (creator ID + fan address).

FAQ

Why do I see "Set API URL in Actions"?

The page reads data from the Unite API. Enter the base URL (e.g. http://127.0.0.1:8765) in the Actions panel and click Save API URL, then refresh.

How do I run the Unite API?

From the directory containing Unite.py run: python Unite.py serve --port 8765. Then open this page and set the API URL to http://127.0.0.1:8765 (or your host/port).

Can I create listings or place offers from this page?

This interface supports viewing creators, collectibles, listings, and offers, plus registering creators and following via the Actions panel. For creating listings, placing offers, filling, or cancelling, use the Unite CLI (e.g. python Unite.py create-listing ...) or interact with the SiamsoProtocol contract directly.

What is SiamsoProtocol?

SiamsoProtocol is an EVM contract for a content-creator and collectible fan site and exchange: creators register, mint collectibles, set royalties and allowlists; fans follow creators and trade collectibles via an on-chain listing and offer book with protocol fees.

Query parameters

You can pass ?api=http://127.0.0.1:8765 to set the API base URL, and ?tab=collectibles to open a specific tab (creators, collectibles, listings, offers, actions).

Mint collectible (POST)

If your Unite API supports POST /api/mint, you can mint a new collectible here. Otherwise use the Unite CLI.

Unite CLI reference

Commands you can run with python Unite.py:

  • register-creator --account 0x... --handle my_handle — Register as creator.
  • mint --creator-id creator_1 --account 0x... --to 0x... --supply-cap 100 — Mint collectible.
  • follow --creator-id creator_1 --fan 0x... — Follow a creator.
  • list-creators, list-collectibles, list-listings, list-offers — List data.
  • create-listing --collectible-id col_1 --seller 0x... --amount 1 --price-wei 1000000 — Create sell listing.
  • place-offer --collectible-id col_1 --bidder 0x... --amount 1 --price-wei 1000000 — Place buy offer.
  • serve --port 8765 — Start API server for this web UI.
  • stats, export, import, creator-stats, collectible-stats — Utilities.

Data schema (Unite API)

Response shapes used by this UI:

GET /api/stats

{ "creators": number, "collectibles": number, "listings": number, "offers": number, "fan_follows": number }

GET /api/creators

{ "creators": [{ "creator_id", "handle", "account", "active", "registered_at", "follower_count" }], "count": number }

GET /api/collectibles

{ "collectibles": [{ "collectible_id", "creator_id", "content_hash", "supply_cap", "total_minted", "frozen" }], "count": number }

GET /api/listings

{ "listings": [{ "listing_id", "collectible_id", "seller", "amount", "price_wei", "expires_at" }], "count": number }

GET /api/offers

{ "offers": [{ "offer_id", "collectible_id", "bidder", "amount", "price_wei", "expires_at" }], "count": number }

POST /api/register-creator

Body: { "account": string, "handle": string, "content_root"?: string }

POST /api/follow

Body: { "creator_id": string, "fan": string }

POST /api/mint

Body: { "creator_id", "account", "to", "supply_cap"?, "content_hash"?: string }

SiamsoProtocol contract overview

The on-chain contract (EVM) provides:

  • Roles: Curator (admin), Steward (fee/config), Guardian (pause). Addresses are set at deploy; no constructor args required.
  • Creators: Register with content root and handle; update content root; can be deactivated by curator.
  • Collectibles: Minted by creators with supply cap; transferable; optional royalty (bps) and allowlist per collectible.
  • Fan follows: Any address can follow/unfollow a creator (on-chain or via indexer).
  • Listings: Sellers list collectible amount at price in wei; buyers fill with ETH; protocol fee and optional royalty deducted.
  • Offers: Bidders lock ETH for amount at price; sellers accept and receive ETH minus fee/royalty.
  • Safety: Reentrancy guard, pause by guardian, fee/royalty caps, EIP-55 addresses, immutable role addresses.

Unite mirrors this logic in local state and REST API for development and testing; production flows can use the contract directly.

Glossary

Creator
An address that has registered with a handle and content root; can mint collectibles and set royalties/allowlists.
Collectible
A tokenized item bound to a creator, with supply cap and optional royalty; fungible by amount (balance per address).
Listing
A sell order: seller offers amount of a collectible at a price in wei; buyer fills with ETH.
Offer
A buy order: bidder locks ETH for amount at price; seller accepts and receives ETH, bidder receives collectible.
Fan follow
An address following a creator (stored on-chain or in Unite state); used for discovery and gating.
Unite
Single-file Python app: local state, CLI, and REST API for SiamsoProtocol; companion to this web UI.
BasedCollectiveFans
This web interface: view creators, collectibles, listings, offers; register creators and follow via API.

Step-by-step: Connect Unite and view data

  1. Install Python 3 if needed. Download or clone the project that contains Unite.py.
  2. Open a terminal in that directory and run: python Unite.py serve --port 8765. You should see "Serving Unite API at http://127.0.0.1:8765".
  3. Open this HTML file (BasedCollectiveFans) in a browser. Go to the Actions tab.
  4. In "API base URL", enter http://127.0.0.1:8765 (or http://localhost:8765) and click "Save API URL".
  5. Click "Refresh" in the header (or switch tabs). The stats should update and creators/collectibles/listings/offers will load if any exist.
  6. To add test data: in Actions, fill "Account" and "Handle" and click "Register creator". Then refresh. Optionally use "Mint collectible" or the Unite CLI to mint, then create listings or offers via CLI.

Troubleshooting

CORS errors in browser console

If the Unite server runs on a different origin than the file:// or site hosting this page, the browser may block requests. Run Unite with --host 0.0.0.0 if needed and ensure the API URL matches the address you use to open this page. For local file opening, use the same host (e.g. 127.0.0.1) in the API URL.

"Failed to load creators" or empty lists

Confirm the API base URL is correct and the Unite server is running. Open the API URL in a new tab (e.g. http://127.0.0.1:8765/api/creators) and check that JSON is returned. If the server was just started, the state may be empty until you register a creator via Actions or the CLI.

Export CSV is empty

Export uses the current in-memory cache (creators or collectibles loaded in the active tab). Switch to the Creators or Collectibles tab, click Refresh, then click Export CSV. If no data was loaded, the CSV will have only headers or one row.

Browser support

This page uses standard ES5+ JavaScript, fetch, and localStorage. Supported in modern Chrome, Firefox, Safari, and Edge. Query parameters and API URL are stored in localStorage for the current origin.

Contract addresses (deploy-specific)

SiamsoProtocol uses four immutable addresses set in the constructor: curator, steward, guardian, and fee recipient. They are EIP-55 checksummed and set at deploy time. This web UI does not display or require contract addresses; it talks only to the Unite API. To interact with the live contract, use a wallet or script with the deployed SiamsoProtocol address on your chain.

Version and file info

BasedCollectiveFans is a single-file static web app. No build step is required; open index.html in a browser. The Unite API (Python) must be running separately for data. SiamsoProtocol is a Solidity contract deployable to any EVM chain; this UI does not connect to the chain directly.

Privacy and data

This page stores only the API base URL (and optional query params) in your browser's localStorage. No creator, collectible, or wallet data is stored by the page. All data displayed is fetched from the Unite API you configure. If you use a remote Unite server, its operator could log requests. For production, deploy Unite and this UI on infrastructure you control or trust.
